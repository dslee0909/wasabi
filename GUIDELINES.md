## 3.5. 차별화 기능 상세 명세 ⭐

### A. 임시 음성채널 (Join-to-Create)

- **트리거 채널**(예: `➕ 방 만들기`)에 입장하면 봇이 개인 음성채널을 생성하고 사용자를 이동시킴
- 채널이 **비면 자동 삭제**
- 방 이름/인원 제한은 생성자가 조절 가능(추후)

### B. 게임 파티 모집

1. 사용자가 **구인구직 채널**에 정해진 **양식**으로 채팅 입력
   - 예: `@디스코드게임태그 게임이름 (인원수) 디스코드카테고리`
2. 봇이 해당 메시지에 **스레드를 자동 생성**
3. 스레드 안에 봇이 **안내 문구 + `참여` 버튼** 메시지 게시
4. 사용자가 `참여` 버튼 클릭 → 봇이 스레드에 **`{닉네임} 참여`** 채팅을 자동 생성

> ✅ 확정: 참여는 **버튼 클릭 → 자동 메시지 생성** 방식 (모달 입력 아님)

### C. 관전 모드 버튼 (요굴봇 참고)

- 버튼 클릭 → 서버 닉네임 앞에 **`[관전]`** 접두사 부착(토글: 다시 누르면 해제)
- 봇 역할이 대상보다 **위**에 있어야 닉네임 변경 가능 (서버 소유자 닉은 변경 불가 — 예외 처리)

### D. 활동시간 기반 레벨/승급 시스템

- **집계 기준: 음성채널 접속 시간만** (확정) — 게임 활동(Presence) 감지는 사용 안 함
  - 이유: 유저가 게임 상태 공유를 꺼두면 봇이 게임 실행을 감지할 수 없음 → 예측 가능한 음성 시간 기준으로 통일
- 접속 시간을 경험치로 환산 → 레벨업
- **집계 기간**: 최근 1개월(롤링 윈도우) 기준 — 신입 → 정규 등급 **승급 관리용**
- **집계 제외 규칙** (핵심):
  - `C`의 **`[관전]`** 접두사가 닉네임에 있으면 → 시간·경험치 **미집계**
  - 임시 음성채널 중 **지정된 일부 채널**은 관전 여부와 무관하게 **미집계** (제외 채널 목록으로 관리)

## 5. 보안 원칙 (⚠️ 가장 중요)

1. **봇 토큰은 절대 코드에 직접 쓰지 않는다** → `.env` 파일에 보관
2. `.env`**는 Git에 커밋하지 않는다** → `.gitignore`에 반드시 추가
3. 토큰이 유출되면 즉시 Discord 개발자 포털에서 **재발급(Reset)**
4. 필요한 **권한(Intents)만** 활성화

---



## 6. 개발 환경 준비 순서

1. Discord 개발자 포털에서 애플리케이션 & 봇 생성 → 토큰 발급
2. **Privileged Gateway Intents** 활성화 (Server Members, Message Content)
3. 봇을 내 서버에 초대 (OAuth2 URL, `bot` 스코프 + 필요 권한)
4. Python 가상환경 생성 후 `pip install discord.py python-dotenv`
5. `bot.py` 작성 → 실행 → 온라인 확인
