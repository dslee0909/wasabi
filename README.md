# 디스코드 봇 — 기능 정리

게임/친목 커뮤니티 서버용 관리·자동화 봇. Python + discord.py.

---

## 실행 방법

```bash
# 1) 의존성 설치 (최초 1회)
pip install -r requirements.txt

# 2) .env 파일에 토큰 넣기 (DISCORD_TOKEN=...)

# 3) 실행
python bot.py
```

> 봇이 켜져 있어야 작동해요. PC를 끄면 오프라인이 됩니다.

## 기능 상세

### 1. 환영 / 퇴장 (자동)
- 새 멤버 입장 → 환영 임베드 자동 전송
- 멤버 퇴장 → 퇴장 로그
- 채널: 서버의 '시스템 메시지 채널'

### 2. 역할 자동화
- **자동 역할**: 입장 시 지정한 역할 자동 부여
- **반응 역할**: 선택판의 이모지 클릭 → 역할 받기 / 떼기
- **수동 부여/회수**: 관리자가 특정 멤버 역할 관리

### 3. 임시 음성채널 (Join-to-Create)
- `➕ 방 만들기` 채널 입장 → 개인 방 자동 생성 + 이동
- 방이 비면 자동 삭제 / 생성자가 자기 방 방장

### 4. 관전 모드
- 버튼으로 닉네임 앞에 `[관전]` 토글
- `[관전]` 상태면 음성 시간 미집계
- (서버 주인 닉네임은 디스코드 규칙상 변경 불가)

### 5. 게임 파티 모집
- 구인구직 채널에 **`@역할`로 시작하는 글** → 스레드 자동 생성
- 스레드에 `참여`/`취소` 버튼
- `참여` → 웹훅으로 본인 아바타·닉네임으로 참여 메시지 (본인이 친 것처럼)
- `취소` → 본인 참여 메시지 삭제

### 6. 활동시간 레벨 (음성 기반)
- 음성채널 접속 시간 집계 → 레벨 (`[관전]`·제외채널은 미집계)
- **레벨 = 영구** (전체 누적, 절대 안 내려감)
- 곡선 조정 가능: `레벨 n = 기준시간 × n^곡선` 시간 필요

### 7. 승급 / 잠수 관리 (최근 활동 기반)
- **자동 승급**: 특정 레벨 도달 시 역할 자동 부여 (5분마다 확인)
- **잠수 확인**: 최근 N일 활동 적은 멤버 목록 (자동 추방 X, 관리자 판단)

### 8. 스탯 카드 (`/레벨`)
- 레벨, 음성 활동(1/7/30일), 메시지(1/7/30일), 음성·메시지 순위
- 메시지는 `on_message`로 수집 (채널·시각 기록)

## 파일 구조

```
Bot/
├─ bot.py              # 진입점 (cogs 자동 로드, /ping)
├─ store.py            # config.json 설정 저장/불러오기
├─ voicetime.py        # DB·시간·레벨·순위 공용 함수
├─ config.json         # 서버별 설정 (자동역할, 트리거채널 등)
├─ data.db             # SQLite (voice_sessions, messages)
├─ requirements.txt
├─ .env                # 봇 토큰 (커밋 금지)
└─ cogs/
   ├─ welcome.py       # 환영/퇴장 + /환영
   ├─ roles.py         # 역할 자동화 + /역할
   ├─ reactionroles.py # 이모지 역할 + /반응역할
   ├─ tempvoice.py     # 임시 음성채널 + /음성
   ├─ spectate.py      # 관전 버튼 + /관전
   ├─ party.py         # 파티 모집 + /파티
   ├─ leveling.py      # 음성시간 추적·레벨·스탯카드 + /레벨 /순위 /레벨설정
   └─ activity.py      # 승급·잠수 + /활동
```

## 데이터 저장

- **config.json** — 서버별 설정 (자동역할 ID, 트리거 채널, 리액션 패널, 레벨 곡선, 승급 기준 등)
- **data.db** (SQLite)
  - `voice_sessions` — 음성 세션 기록 (guild, user, seconds, ended_at)
  - `messages` — 메시지 기록 (guild, user, channel, ts)

## 예정 기능 설계 (확정, 미구현)

### 낚싯대 강화 시스템
- **목적**: 코인 소비처 + 강화 실패의 재미 + `+N` 훈장 (효과는 미미/코스메틱 → 밸런스 안 깨짐)
- **효과**: 강화 +1당 아주 작은 보너스(예: 코인 +1.5%, 상한 있음) 또는 순수 코스메틱
- **승계 규칙**: 티어 업그레이드 시 강화수치 **유지(승계)**
- **티어별 강화 상한** (꼼수 방지 — 높은 티어일수록 더 높이 강화):
  - 나무 +2 · 강철 +4 · 황금 +6 · 다이아 +8 · 용왕 +10 · 전설 +12
- **강화 판정**: 코인 소비 + 확률 성공 (레벨 오를수록 성공률↓). 실패 시 코인만 소모(레벨 유지, 안전판)
- **표시**: `전설의 낚싯대 +7` 처럼 이름 옆 `+N` / 낚시 메시지·상태창에 표기
- **비주얼**: 강화 높을수록 이미지에 글로우 이펙트 (봇이 기존 아트에 렌더 → 강화용 이미지 불필요)
- **구현 시**: balances 에 `rod_enhance` 컬럼 추가, `/강화` 명령, `/낚시대`(상태창) 명령

### 미끼 시스템 (계획 — 영구제)
- **방향 변경**: 소모품 → **영구제** (낚싯대처럼 한 번 사면 계속 적용)
- 낚싯대와 **별도 슬롯**으로 장착 (예: 지렁이/반짝미끼/자석 등), 효과는 낚싯대 효과와 합산
- 소모품이 아니라 인벤토리·사용횟수 관리 불필요 (balances 에 `bait` 컬럼 등으로 단순 저장)
- ⚠️ 영구 효과라 낚싯대와 스택되면 오버밸런스 주의 → 미끼 효과는 작게/특수하게 (반짝이 확률, 특정 물고기 확률 등)
